Overview

What Is This Project?

A confidence-aware longitudinal MRI analysis pipeline for comparing the same subject across different time points.

Research Purpose

The project compares baseline and follow-up brain MRI scans to inspect lesion and tissue changes over time. Its first focus is multiple sclerosis progression inspection, while the segmentation-based design makes it extensible to other neurological diseases.

This is not a clinical diagnostic tool; it is a research-oriented inspection and analysis pipeline.

Pair Scans

Baseline and follow-up NIfTI scans are paired around the same subject to support longitudinal comparison.

Analyze Change

Preprocessing, masking, registration, template construction, and voxelwise change analysis turn scan pairs into inspectable outputs.

Score Confidence

NCC, MAE, Dice, brain volume, ROI agreement, and model cross-checks help separate useful findings from preprocessing or registration artifacts.

Problem & Motivation

Why Longitudinal MRI Comparison Is Difficult

Multiple sclerosis can cause subtle lesion and tissue changes over time, but those changes are difficult to separate from scanner, modality, preprocessing, and registration effects.

Disease signal

Subtle changes over time

MS-related lesion and tissue changes may be small, irregular, and distributed across different brain regions.

Technical noise

Scanner and modality differences

Reliable comparison requires intensity normalization, brain masking, spatial alignment, and quality-control signals.

Extensibility

Segmentation-based analysis

A segmentation-aware workflow makes the system easier to extend to additional neurological disease research scenarios.

MRI Modalities

T1w, T2w, and FLAIR Perspective

The project is designed around a multimodal MRI perspective, because MS-related changes may appear differently across T1w, T2w, and FLAIR scans.

T1w

Provides anatomical structure and tissue contrast. Useful for registration, template construction, and anatomical comparison.

T2w

Highlights abnormal tissue signal and supports lesion burden inspection.

FLAIR

Suppresses cerebrospinal fluid signal, making periventricular and juxtacortical MS lesions easier to inspect.

Core Pipeline

From Scan Pair to Change Inspection

The core flow is not limited to a CPU-only demo. Local CPU mode validates the essential imaging steps, while full analysis extends the same queue with heavier segmentation and confidence stages.

01

Preprocessing

Baseline and follow-up scans are prepared through resampling, filtering, and consistent output organization.

02

Brain Mask Extraction

Brain masks are extracted to isolate comparable brain tissue and reduce background-driven artifacts.

03

Masked Intensity Normalization

Intensity normalization is computed inside the shared brain region to make time-point comparison more reliable.

04

Direct / Optimized Registration

Scans are spatially aligned so baseline and follow-up anatomy can be compared voxel by voxel.

05

Subject-Specific Template

A within-subject template supports longitudinal comparison in a stable subject-centered space.

06

Voxelwise Change Analysis

Difference maps, absolute difference maps, and change-direction outputs are produced for inspection.

07

Report Generation

NIfTI outputs, CSV metrics, JSON QC reports, and pipeline_step_report.json summarize the analysis.

Execution Modes

Local CPU + Full Analysis GPU

The advanced path is an optional execution route for heavier AI-ready medical imaging stages, not a requirement for every lightweight validation run.

LOCAL_CPU

Local CPU Mode

Runs on standard hardware for lightweight validation of preprocessing, brain masking, normalization, registration, template construction, and voxelwise reports.

FULL_ANALYSIS

Full Analysis GPU Mode

Targets SynthSeg segmentation, FastSurfer cross-checking, ROI registration QC, model agreement, confidence scoring, and MS lesion detection through the GPU worker queue.
